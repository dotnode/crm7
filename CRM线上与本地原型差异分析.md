# CRM 线上站点与本地 H5 原型差异分析

分析日期：2026-06-23

## 分析范围与方法

- 已启动本地原型：`http://localhost:9090`
- 已用浏览器访问线上站点：`https://crm.xiaoman.cn/`
  - 线上可直接进入已登录工作台：`/dashboard`
  - 已抽样浏览线上客户列表：`/crm/customer/list`
- 并行拆分 20 个页面/模块审查任务，对照：
  - 线上浏览到的真实页面结构
  - `crm_xiaoman_prototype_spec.md`
  - 本地实现 `js/app.js`、`index.html`、`css/style.css`

> 说明：本仓库是静态 H5 原型，部分深层页面为空态在 `CLAUDE.md` 中已标注为已知范围。`OKKI AiReach`、`OKKI Leads` 为业务上刻意移除/隐藏的入口，不纳入缺失项。本文其余部分按“与线上/spec 是否一致”记录差异，便于后续决定是否补齐。
>
> 记录口径：凡是指出“布局不正确 / 布局被简化 / 结构不一致”的地方，必须同时写明“正确布局应为”，避免只描述问题、不说明目标形态。

## 总体结论

本地原型已经覆盖了 OKKI CRM 的主导航、hash 路由、主要列表页、详情页、抽屉/弹窗骨架和大量静态样例数据；但与线上真实站点和 spec 相比，仍存在以下主要差异：

1. **全局静态入口较多**：顶栏按钮、全局搜索和若干模块入口只保留视觉外观，缺少下拉、面板、跳转或联动交互。
2. **多个页面是“外观壳”而非完整交互**：顶部按钮、筛选、导入导出、列设置、批量操作、下拉菜单、tab 内容切换大量未绑定实际行为。
3. **复杂页面布局被简化**：邮件三栏阅读、客户/线索客群树、商机列表/看板切换、订单详情 tab、工作报告弹窗等均与线上/spec 有明显压缩。
4. **大量 tab 只有高亮，没有内容切换**：企业管理销售配置、订单详情、沟通页、部分详情页等只有 `renderTabs()` 或静态按钮，缺少对应 panel。
5. **企业管理、协同、团队数据模块占位偏多**：不少页面只有一个表格或空态，未覆盖完整设置项、规则配置、报表、弹窗与跳转链路。

### 1. 顶栏按钮大多是静态

本地 `TM / 消息 / 客服 / 帮助 / 头像` 是静态按钮，未接入下拉、消息面板、帮助入口或账号菜单。顶部搜索也没有随当前模块联动。

证据：
- `index.html:16-27`
- `crm_xiaoman_prototype_spec.md:37-45`

### 2. 已排除项：`OKKI AiReach` 与 `OKKI Leads`

`OKKI AiReach`、`OKKI Leads` 是业务上刻意移除/隐藏的入口，不作为本次差异分析中的功能缺失项。若后续业务要求恢复，再单独评估导航和二级菜单还原范围。

## 技术性问题与实现风险

### 1. tab 组件大量只高亮不换内容

部分页面只有 `renderTabs()` 或 toggle 按钮，没有配套 `renderTabPanels()` / 内容切换逻辑。典型页面：

- 沟通页
- 销售订单详情
- 企业管理客户设置
- 企业管理线索设置
- 企业管理商机设置
- 产品设置

证据：
- `js/app.js:248-267`
- `js/app.js:937-971`
- `js/app.js:3258-3283`
- `js/app.js:3793-3808`
- `js/app.js:3839-3862`
- `js/app.js:3906-3918`

### 2. 多个按钮只有视觉，没有动作

常见包括：

- 顶栏 `TM / 消息 / 客服 / 帮助 / 头像`
- 工作台 `汇率计算器 / 工作台配置 / 显示设置 / 全部已读`
- 邮件 `高级搜索 / 附件 / 模板 / 快速文本 / 产品 / 单据 / 收款`
- 沟通 `营销数据 / 社媒管理 / 使用指南`
- 订单 `关联生成 / 发起任务`
- 任务 `任务完成情况 / 任务设置`

### 3. 部分路由断链或语义不一致

- 审批页跳 `collab-approval-flow`，但缺对应 renderer。
- `business-ai-analysis` 作为侧栏子页存在，但实际复用 `business-list`，与 spec “分析面板”语义不一致。
- 订单详情多个 tab 未对应真实页面或 panel。

## 建议补齐优先级

### 优先级 A：影响主流程走查

1. 补齐顶栏按钮、全局搜索和常用入口的下拉/面板/跳转联动。
2. 补齐邮件三栏阅读和写邮件关键动作。
3. 补齐客户/线索列表的筛选、批量操作、客群/AI 客群切换。
4. 补齐商机列表的列表/看板视图与完整阶段。
5. 补齐订单详情 tab、银行信息和 `关联生成 / 发起任务` 下拉。

### 优先级 B：影响页面一致性

1. 统一页面标题与 spec：如客户列表应为“客户管理”。
2. 补齐 tab panel，避免只切高亮不切内容。
3. 补齐客户详情、线索详情、商机详情的字段和右侧关联卡片。
4. 补齐工作台按钮弹窗与任务操作闭环。

### 优先级 C：深层配置和占位页

1. 企业管理销售配置的多 tab 内容。
2. 产品设置、任务设置、审批流管理、工作报告完整弹窗。
3. 团队数据报表/AI 数据页面的图表和历史区。
4. 云盘、回收箱、自定义报表等深层页面。
