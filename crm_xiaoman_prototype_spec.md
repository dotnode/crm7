# OKKI CRM 页面与功能原型说明文档

> 采集方式：使用真实浏览器访问 `https://crm.xiaoman.cn/`，基于当前登录账号可访问范围整理。  
> 目标：阅读本文即可还原主要信息架构、页面布局、字段、按钮、筛选项、列表列名、抽屉/表单结构与关键交互。  
> 注意：本文只保留页面结构与字段名，不记录真实客户、邮件、联系人等业务数据。

## 1. 全局框架

### 1.1 页面整体布局

所有页面采用统一后台管理系统布局：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部栏：Logo / 全局搜索 / TM / 消息 / 客服 / 帮助 / 用户头像 │
├──────────┬───────────────────────────────────────────────────┤
│ 一级侧栏 │ 二级模块菜单 + 当前页面主体内容                  │
│ 常用     │                                                   │
│ 工作台   │                                                   │
│ 邮件     │                                                   │
│ 沟通     │                                                   │
│ 线索     │                                                   │
│ 客户     │                                                   │
│ 商机     │                                                   │
│ OKKI Leads│                                                  │
│ 交易     │                                                   │
│ 团队数据 │                                                   │
│ 协同     │                                                   │
│ 企业管理 │                                                   │
└──────────┴───────────────────────────────────────────────────┘
```

### 1.2 顶部栏

| 区域 | 内容 | 说明 |
|---|---|---|
| Logo | OKKI / 小满标识 | 点击返回首页/工作台。 |
| 全局搜索 | 搜索类型下拉 + `请输入搜索关键字` 输入框 | 搜索类型会随当前模块显示，如客户、线索、商机、成员等。 |
| TM | `TM` 按钮 | 进入或唤起阿里 TM 沟通相关功能。 |
| 消息 | `消息` 按钮，角标 `99+` | 打开系统消息/通知。 |
| 客服 | `客服` 按钮 | 打开在线客服入口。 |
| 帮助 | `帮助` 按钮 | 打开帮助入口。 |
| 用户区 | 头像/用户入口 | 用于账号、个人设置等。 |

### 1.3 一级导航

| 一级导航 | 页面/模块定位 |
|---|---|
| 常用 | 用户常用入口管理。 |
| 工作台 | 个人待办、AI 跟进任务、日程、目标概览等。 |

| 沟通 | WhatsApp、Facebook、Instagram、网站聊天、TM 等沟通渠道。 |
| 线索 | 线索列表、线索智能孵化。 |
| 客户 | 客户列表、公海客户、建档建议、客户查重、客户设置。 |
| 商机 | 商机列表、AI 商机转化分析、AI 自动化配置。 |
| OKKI Leads | 一站式获客、渠道获客、动态监测、触达工具。 |
| 交易 | 跟单流程、产品管理、订单管理、资金管理。 |
| 团队数据 | 目标完成情况、目标管理、团队墙、AI 数据洞察、统计报表等。 |
| 协同 | 日程、审批、云盘、通讯录、任务。 |
| 企业管理 | 基础管理、销售管理、交易管理、沟通管理、外部对接、流程规则、数据管理。 |

## 2. 工作台

### 2.1 页面定位

URL：`/dashboard`  
页面标题：工作台

工作台用于个人首页和日常待办聚合，强调 AI 推荐任务、日程视图和快捷工具。

### 2.2 页面布局

```text
工作台主体
├─ 欢迎区
│  ├─ Hi + 用户名
│  ├─ 时区/时间显示
│  ├─ 汇率计算器按钮
│  └─ 工作台配置按钮
├─ 日程区
│  ├─ 日程按钮
│  ├─ 周 / 月 / 列表 视图切换
│  └─ 上一段 / 今天 / 下一段 等导航按钮
└─ 跟进任务区
   ├─ 标题：跟进任务
   ├─ 显示设置
   ├─ 说明：基于 AI 分析与管理者配置，为你推荐以下跟进任务。立即行动，持续提升业绩！
   ├─ 任务卡片列表
   └─ 卡片操作：生成话术 / 生成邮件 / 回复 / 忽略 / 全部忽略 / 全部已读 / 立即配置
```

### 2.3 主要功能

| 功能 | 说明 |
|---|---|
| 汇率计算器 | 快速换算外贸常用货币。 |
| 工作台配置 | 配置工作台模块展示。 |
| 日程视图 | 支持周、月、列表视图切换。 |
| 跟进任务 | AI 和管理规则推荐客户/线索/邮件等跟进事项。 |
| 显示设置 | 配置跟进任务显示内容。 |
| AI 辅助动作 | 根据任务生成话术或邮件。 |
| 忽略任务 | 单条忽略或批量忽略。 |

```text
├─ 移入公海白名单
├─ 公海领取规则
├─ 公海显示设置
├─ 公海分组

```

| 区域 | 字段/操作 |
|---|---|
| 移入公海规则 | 从模板添加、添加规则、提前 N 天发送邮件通知、启用状态、规则名称、移入条件、生效客群、开始时间、操作。 |

#### 12.5.5 客户跟进规则 Tab

客户跟进规则定义哪些行为会刷新客户时间字段，从而影响掉公海和任务截止时间。

| 规则组 | 可选行为 |
|---|---|
| 影响“最近联系时间” | 获取公海/同事客户、发送邮件、接收邮件、营销邮件、编辑客户、新建跟进、新建/编辑商机、新建/编辑报价单、新建/编辑销售订单、上传客户文档、发送聊天消息、接收聊天消息。 |
| 影响“最近跟进时间” | 发送聊天消息、发送邮件、营销邮件、新建跟进。 |
| 操作 | 全选、编辑。 |

聊天渠道包含 TM、WhatsApp、FB Messenger、WhatsApp 企业版、Instagram Messenger、企业微信；邮件和跟进行为同时覆盖客户和商机对象。

#### 12.5.6 客户成交规则 Tab

| 规则项 | 说明 |

| 计入绩效订单的订单日期 | 多个符合默认绩效“成交订单金额”的订单时，取最新的订单日期为最近成交日期。 |

#### 12.5.8 客户上限 Tab

|---|---|

| 自动化创建超限规则 | 继续创建且归属于原跟进人；继续创建且归属于指定账号；继续创建且不归属任何成员并进入指定公海分组；不创建。 |

## 13. 页面原型共性规范

### 13.1 列表页通用结构

```text
列表页
├─ 页面标题 + 视角/范围切换
├─ 主操作按钮：新建 / 导入 / 导出 / AI 助理 / 设置
├─ 左侧分类或分组树（如客群、状态、来源）
├─ 右侧数据区
│  ├─ 统计数量
│  ├─ 筛选条件
│  ├─ 表格或看板
│  └─ 分页
└─ 行级操作
```

### 13.2 新建抽屉通用结构

```text
右侧抽屉
├─ 标题
├─ 提示条（可选）
├─ 主要信息分组
├─ 显示设置
├─ 必填字段以 * 标识
├─ 展开全部（选填）
├─ 关联信息分组（联系人/附件/产品等）
└─ 底部固定按钮：取消 / 确定并继续新建 / 确定
```

### 13.3 表格通用能力

| 能力 | 说明 |
|---|---|
| 批量选择 | 表头和行首有复选框。 |
| 排序 | 时间、名称等字段支持排序图标。 |
| 自定义列 | 多数列表页有列设置入口。 |
| 分页 | 共 N 条、页码、每页条数、跳至页。 |
| 行级跳转 | 名称字段通常可点击进入详情。 |
| 行级操作 | 领取、编辑、删除、匹配、生成等。 |

### 13.4 AI 能力入口

| 位置 | AI 功能 |
|---|---|
| 工作台 | AI 推荐跟进任务、生成话术、生成邮件。 |

| 客户详情 | AI 客户分析、AI 背调。 |
| 商机 | AI 商机转化分析、AI 商机自动化配置、自动补录历史商机。 |

| 团队数据 | AI 数据监测、AI 数据分析、AI 批量质检。 |

## 14. 原型绘制建议

### 14.1 推荐优先级

1. 先画全局框架：顶部栏 + 一级侧栏 + 二级菜单 + 内容区。
2. 再画核心业务闭环：线索列表 → 新建线索 → 客户列表 → 新建客户 → 客户详情 → 新建商机 → 商机看板。
3. 再画沟通与邮件闭环：收件箱 → 写邮件 → 客户详情写邮件入口。
4. 再画交易与协同：跟单流程、产品、阿里商品、日程、任务。
5. 最后画企业管理配置页：功能开关、角色权限、自定义字段。

### 14.2 关键页面原型组件清单

| 组件 | 需要复用的页面 |
|---|---|
| 顶部全局搜索栏 | 所有页面。 |
| 一级导航侧栏 | 所有页面。 |
| 二级菜单树 | 线索、客户、交易、团队数据、协同、企业管理。 |
| 左侧分类树 | 线索列表、客户列表、公海客户、产品。 |
| 数据表格 | 线索、客户、公海、产品、阿里商品、目标排行、日程、通讯录、任务、角色权限、自定义字段。 |
| 右侧抽屉表单 | 新建线索、新建客户、新建商机。 |
| AI 提示横幅 | 商机、产品、工作台。 |
| 详情页信息卡 | 客户详情。 |
| 富文本编辑器 | 写邮件。 |
| 流程图/泳道图 | 跟单流程。 |
